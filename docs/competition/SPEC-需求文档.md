# BeeFintech 官网升级项目 SPEC 需求文档

> 生成方式：使用 `superpowers:brainstorming` 技能梳理产品定位、功能边界、用户路径与验收标准。

## 1. 项目背景

本项目面向公司「Vibe Coding 实战成长赛」，目标是将 BeeFintech 官网从静态品牌展示页升级为可交付的保险科技 Web 服务平台。项目围绕保险经纪业务数字化场景，结合 AI 辅助设计、前端实现、后台管理与数据库服务，形成一个可演示、可扩展、可沉淀的官网升级样板。

当前官网主要承担品牌宣传与产品介绍职能。本次升级在保留 BeeFintech 品牌识别与淡蓝色金融科技视觉风格的基础上，新增动态内容、预约演示、FAQ 自助查询、保险科技资讯和管理后台，使网站具备基础服务平台能力。

## 2. 产品定位

BeeFintech 官网升级版定位为：

面向香港保险经纪公司、保险代理团队和保险科技生态伙伴的一站式保险科技服务门户。

网站不做 C 端保险销售，也不走重云厂商风格，而是采用轻量 B2B 金融科技风格，突出以下价值：

- 连接保险公司、经纪渠道与客户服务。
- 展示保险经纪业务管理系统与 LifeBee APP 移动拓客工具。
- 提供 FAQ、保险科技资讯、预约演示等服务入口。
- 支持后台管理 FAQ、资讯与预约记录，形成前后台闭环。

### 4.2 FAQ 自助查询

FAQ 功能包含：

- 首页展示 4 条常见问题。
- FAQ 列表页展示全部问题。
- FAQ 搜索页支持关键词搜索。
- 匹配不到内容时展示“无相关问题”。
- FAQ 详情页展示问题与回答。
- 支持从详情页返回 FAQ 列表或首页。

### 4.3 保险科技资讯

资讯功能包含：

- 首页展示 3 条精选资讯。
- 每条资讯包含图片、分类、标题和时间。
- 点击“查看更多”进入资讯列表页。
- 资讯列表页展示每条资讯的图片、分类、标题和时间。
- 点击资讯进入详情页。
- 资讯详情页展示分类、标题、发布时间和正文。

### 4.4 预约演示

预约演示表单字段：

| 字段 | 是否必填 |
| --- | --- |
| 姓名 | 必填 |
| 公司名称 | 必填 |
| 职位 | 必填 |
| 手机号 | 必填 |
| 邮箱地址 | 选填 |
| 关注产品模块 | 选填 |
| 备注 | 选填 |

提交成功后提示：

“已收到您的预约，我们会在一个工作日内联系您。”

预约信息需写入真实数据库，并可在管理后台查看。

### 4.5 管理后台

后台包含登录页和管理页。

后台功能：

1. 密码登录
   - 默认通过环境变量 `ADMIN_PASSWORD` 配置。
   - 登录成功后进入后台管理页。

2. FAQ 管理
   - 查看 FAQ 列表。
   - 新增 FAQ。
   - 编辑 FAQ。
   - 删除 FAQ。

3. 资讯管理
   - 查看资讯列表。
   - 新增资讯。
   - 编辑资讯。
   - 删除资讯。

4. 预约管理
   - 查看预约记录。
   - 字段包括提交时间、姓名、公司、职位、手机号、邮箱、关注产品模块、备注。

## 5. 技术架构

### 5.1 当前本地演示架构

- 前端：HTML / CSS / JavaScript
- 本地服务：Node.js `server.js`
- 本地数据库：SQLite

### 5.2 线上部署目标架构

- 代码托管：GitHub
- 前端部署：Vercel
- API：Vercel Functions
- 数据库：Supabase Postgres
- 后台认证：基于密码和 HTTP Cookie 的轻量登录态

线上部署后访问路径：

- 官网首页：`https://项目名.vercel.app/`
- 管理后台登录：`https://项目名.vercel.app/admin-login.html`

## 9. 非功能需求

1. 视觉风格
   - 使用 BeeFintech 品牌 Logo。
   - 主色调为淡蓝色金融科技风格。
   - 页面整体轻量、清爽、B2B 专业感。

2. 可访问性与交互
   - 顶部导航应可跳转到对应模块。
   - 预约演示按钮应跳转到预约表单。
   - FAQ、资讯点击后进入独立页面，而不是在首页下方展开。

3. 数据安全
   - Supabase `service_role key` 不得暴露到前端。
   - 后台增删改查必须通过服务端 API。
   - 密钥通过 Vercel 环境变量配置，不提交 GitHub。

4. 可部署性
   - 项目代码托管于 GitHub。
   - 线上通过 Vercel 部署。
   - 数据库使用 Supabase。
