# BeeFintech 官网升级项目执行规划文档

> 生成方式：使用 `superpowers:writing-plans` 技能，将已确定需求拆解为可执行任务、交付物、验证方式与风险控制项。

## 1. 项目目标

在 1-2 天内完成 BeeFintech 官网升级 MVP，使其从静态宣传型官网升级为具备服务入口、动态内容、预约演示、内容管理后台与真实数据库支撑的保险科技 Web 服务平台。

核心交付目标：

- 官网首页达到可演示的高保真视觉效果。
- 至少实现 1 项增值服务功能，实际实现 FAQ 自助查询、保险科技资讯、预约演示、后台管理。
- 所有代码托管至 GitHub。
- 支持部署至 Vercel，并通过 Supabase 提供真实数据库能力。
- 输出比赛要求的标准文档。

## 2. 执行原则

1. 先完成可演示闭环，再优化细节。
2. 首页承担品牌展示与业务转化，FAQ/资讯详情放在独立页面。
3. 后台功能以比赛演示为目标，优先保证增删改查链路完整。
4. 线上部署优先选择 Vercel + Supabase，避免本地 SQLite 在公网环境不可持久的问题。
5. 所有密钥通过环境变量配置，不提交到 GitHub。

## 4. 工作拆解

### 阶段一：产品与设计确认

目标：确定官网信息架构、视觉方向和 MVP 范围。

任务：

- 确认网站定位为轻量 B2B 金融科技官网。
- 确认首页导航结构：
  - 产品能力
  - APP 移动端
  - 关于我们
  - 合作伙伴
  - 服务支持
  - 预约演示
- 确认首页模块：
  - Hero 首屏
  - 保险经纪业务管理系统
  - 核心功能模块
  - LifeBee APP 移动拓客工具
  - 关于 BeeFintech
  - 合作伙伴生态
  - 服务支持与预约演示
- 通过 UI 图确认淡蓝色金融科技风格。

产出：

- 首页 UI 方向。
- 页面结构共识。
- MVP 功能清单。

验收方式：

- 用户确认 UI 方向与模块结构。
- 明确不再混入后台 UI 到官网首页。

### 阶段二：首页高保真开发

目标：完成官网首页可演示版本。

任务：

- 实现顶部导航栏和语言切换。
- 实现 Hero 首屏及动态保险数据网络视觉。
- 实现保险经纪业务管理系统 5 个 Tab。
- 实现核心功能模块。
- 实现 LifeBee APP 移动拓客工具 7 个 Tab。
- 实现关于 BeeFintech 模块。
- 实现合作伙伴生态滚动展示。
- 实现服务支持区和预约表单。

涉及文件：

- `index.html`
- `assets/brand/*`
- `assets/system-tabs/*`
- `assets/app-tabs/*`

验收方式：

- 首页在浏览器中可完整滚动浏览。
- 顶部导航可跳转至对应模块。
- UI 风格与已确认长图方向一致。

### 阶段三：FAQ 与资讯页面开发

目标：完成服务支持类增值功能。

任务：

- 首页 FAQ 展示 4 条。
- FAQ 列表页展示全部问题。
- FAQ 搜索页支持关键词精准匹配。
- FAQ 详情页展示问题与回答。
- 首页资讯展示 3 条，包含图片、分类、标题、时间。
- 资讯列表页展示全部资讯。
- 资讯详情页展示完整正文。

涉及文件：

- `faq.html`
- `faq-search.html`
- `faq-detail.html`
- `news.html`
- `news-detail.html`

验收方式：

- FAQ 搜索有结果时显示匹配问题。
- FAQ 搜索无结果时显示“无相关问题”。
- FAQ 和资讯详情页均为独立页面。
- 资讯列表页每条内容均包含图片、分类、标题和时间。

### 阶段四：预约演示与管理后台

目标：形成前后台服务闭环。

任务：

- 预约表单接入 API。
- 提交成功后显示：“已收到您的预约，我们会在一个工作日内联系您。”
- 新增后台登录页。
- 新增后台管理页。
- FAQ 管理支持新增、编辑、删除。
- 资讯管理支持新增、编辑、删除。
- 预约管理支持查看提交时间和预约内容。

涉及文件：

- `admin-login.html`
- `admin.html`
- `api/admin/login.js`
- `api/admin/me.js`
- `api/admin/logout.js`
- `api/bookings.js`
- `api/faqs.js`
- `api/faqs/[id].js`
- `api/news.js`
- `api/news/[id].js`

验收方式：

- 正确密码可进入后台。
- 后台修改 FAQ 后，前台 FAQ 页面读取到更新后的数据。
- 后台修改资讯后，前台资讯页面读取到更新后的数据。
- 前台提交预约后，后台预约管理可看到记录。

### 阶段五：数据库与线上部署

目标：项目支持 GitHub 托管与 Vercel 公网访问。

任务：

- 在 Supabase 执行建表 SQL。
- 在 Vercel 配置环境变量。
- 将项目推送至 GitHub。
- 在 Vercel 导入 GitHub 仓库。
- 验证官网首页、后台登录、FAQ、资讯、预约提交。

涉及文件：

- `docs/supabase-schema.sql`
- `docs/vercel-supabase-deploy.md`
- `.env.example`
- `vercel.json`
- `lib/supabase-rest.js`
- `lib/auth.js`
- `lib/http.js`

Vercel 环境变量：

```text
SUPABASE_URL=Supabase Project URL
SUPABASE_SERVICE_ROLE_KEY=Supabase service_role key
ADMIN_PASSWORD=后台登录密码
ADMIN_SESSION_SECRET=随机长字符串
```

验收方式：

- `https://项目名.vercel.app/` 可访问官网。
- `https://项目名.vercel.app/admin-login.html` 可访问后台登录。
- 后台 CRUD 能影响前台内容。
- 预约提交后 Supabase `bookings` 表出现记录。

## 6. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| Vercel 不支持本地 SQLite 持久化 | 后台数据无法长期保存 | 改用 Supabase Postgres |
| Supabase 密钥泄露 | 数据安全风险 | 仅放入 Vercel 环境变量，不提交 GitHub |
| UI 图与前端还原存在差距 | 影响视觉评分 | 对复杂视觉使用图片资产，对交互区用前端实现 |
| 后台功能范围过大 | 影响 1-2 天交付 | 后台只做 FAQ、资讯、预约管理三类核心功能 |
| 语言切换覆盖不完整 | 影响体验 | 重点覆盖导航、首页主要标题和 CTA |
| 免费部署冷启动或网络波动 | 演示不稳定 | 比赛前提前打开页面预热，并准备本地演示备用 |

### 7.1 首页测试

- 打开首页，检查 Logo、首屏、导航、按钮、模块展示是否正常。

- 点击预约演示，确认跳转到表单。

### 7.2 FAQ 测试

- 首页 FAQ 点击进入详情页。
- 点击“查看更多”进入 FAQ 列表。
- 输入关键词搜索，有匹配时展示结果。
- 输入不存在关键词，展示无相关问题。

### 7.3 资讯测试

- 首页展示 3 条资讯。
- 点击资讯进入详情页。
- 点击“查看更多”进入资讯列表页。
- 资讯列表页每条包含图片、分类、标题、时间。

### 7.4 预约测试
